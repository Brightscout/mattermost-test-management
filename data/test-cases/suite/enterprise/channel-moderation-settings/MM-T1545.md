## MM-T1545: Post Reactions option removed for Guests & Members in System Scheme

**Objective**

> <article>Post Reactions option removed for Guests &amp; Members in System Scheme</article>

**Precondition**

> <article>None</article>

| Name      | Value                                                            |

### Steps

<hr/>

**Step 1**

> <article><ol><li>Login as sysadmin and visit the Permissions page in the system console.</li><li>Edit the System Scheme and remove the Post Reaction option for Guests &amp; Save.</li><li>Visit the Channels page and click on a channel.</li><li>Note that the option to allow Post Reactions to a Guest user should be disabled. A message should be displayed indicating the same.</li><li>Similarly remove the Post Reaction option for Members in System Scheme &amp; Save.</li><li>The option to allow Post Reactions to a Member should be disabled. A message should be displayed indicating the same.</li><li>Login as a Guest User and visit the same channel.</li><li>Check if the user has the permission to react to their posts or to any other posts in the channel.</li><li>Login as a Regular Member and visit the same channel.</li><li>Check if the user has the permission to react to their posts or to any other posts in the channel.</li></ol></article>

**Expected**

> <article>After #4,<br><ul><li>Post Reaction option should be disabled for a Guest User. A message "Post reactions for guests are disabled in the System Scheme" should be displayed.</li></ul>After #6,<br><ul><li>Post Reaction option should be disabled for a Members. A message "Post reactions for guests &amp; members are disabled in the System Scheme" should be displayed.</li></ul>After #8,<br><ul><li>Guest User should not have the permission to react to any post on any channel when the option is removed from the System Scheme.</li></ul>After #10,<br><ul><li>Member should not have the permission to react to any post on any channel when the option is removed from the System Scheme.</li></ul></article>
