## MM-T1454: All menus that open a modal should be announced by the screen reader

**Precondition**

> <article>JAWS Screen Reader required: <a href="https://support.freedomscientific.com/Downloads/JAWS" rel="noopener noreferrer" target="_blank">https://support.freedomscientific.com/Downloads/JAWS</a><br>Eric has a license key to upgrade the free version - ping him if required<br>Switch off the "PC virtual cursor" within JAWS by pressing INSERT+Z<br>Navigation to various focus areas: CTRL+F6 in browsers; F6 in desktop app<br>The feature is not available on the RN apps.</article>

<hr/>

**Step 1**

> <article>All menus that open a modal should be announced by the screen reader<br>–––––––––––––––––––––––––<br>This involves testing all menus that open modals:<ol><li>Check all menu options in the Main Menu</li><li>On a public or private channel that you created (so you have access to all menu options) check all options in the channel dropdown</li><li>In a direct message, check "Edit Header" option in the channel dropdown</li><li>In a group message, check all options in the channel dropdown</li></ol></article>

**Expected**

> <article><ul><li>Each modal that is opened should be announced by the screen reader</li><li>Each sub-section within each modal (e.g. notification preferences modal sub-sections) should be announced as you navigate through the modal in the screen-reader</li></ul></article>
