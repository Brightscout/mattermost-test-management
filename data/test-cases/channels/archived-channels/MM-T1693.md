## MM-T1693: RN apps: If a user manages to open an archived channel in the wrong state, error handling prevents them from updating metadata

**Precondition**

> <article><h1><span style="color: rgb(184, 49, 47);">Important</span></h1>Ensure that `Allow users to view archived channels` is enabled on the server and some public and private archived channels exist<br>(Setting can be found in System Console under "Users and Teams")</article>

### Steps

<hr/>

**Step 1**

> <article>If a user manages to open an archived channel in the wrong state, error handling prevents them from updating metadata<br>–––––––––––––––––––––––––<ol><li>Have another user add you to a channel and make you a channel admin</li><li>Note the unread channel in the drawer</li><li>Set your device to airplane mode</li><li> As the other user, archive the channel</li><li> Disable airplane mode</li><li> Open the channel from the drawer</li><li> Tap on channel header and select "Edit Channel"</li><li>Attempt to edit the channel name, header or purpose</li></ol></article>

**Expected**

> <article><ul><li>An Error is displayed: "<em>You cannot modify an archived channel</em>"</li><li>When you leave the edit page the channel will now show as archived</li></ul></article>
