## MM-T2535: --strict flag

**Objective**

> <article><h1><span style="color: rgb(184, 49, 47);">Include in monthly smoke-test</span></h1></article>

### Details

| Name     | Value                |
| :------- | :------------------- |
| Priority | Low                  |
| Status   | Active               |
| Folder   | mmctl                |

### Steps

<hr/>

**Step 1**

> <article>--strict flag<br>————————————————————————————<ol><li>In a command line window set to the directory where 'mmctl' is installed in</li><li>Connect to a server that is not running the same version as mmctl is (e.g. prev.test with mmctl v5.22)</li></ol></article>

**Expected**

> <article><ul><li>You will get feedback saying "credentials for prev.test stored"</li></ul></article>

**Step Info**
| Name | Value |
| :--- | :--- |
| Fix Versions | v5.22 (April 2020) |
| Related ticket(s) | (<strong><span style="color: rgb(65, 168, 95);">Story</span></strong>) <a href="https://mattermost.atlassian.net/browse/MM-21921">Adding a version check to mmctl commands — MM-21921</a> |

<hr/>

**Step 2**

> <article><ol><li>e.g. "mmctl auth login https://prev.test.mattermost.com --name prev.test --username john.doe --password mysupersecret"<br><br><strong>Note</strong>: Enter valid username and password for a system admin user</li></ol></article>

**Expected**

> <article><ul><li>You will see a warning as follows: "WARNING: server version x.xx.x.release-5.xx.xx.9bb0ce5.af1af0d71b40157b8025d9538a146623.true doesn't match mmctl version 5.22.0"</li></ul></article>

<hr/>

**Step 3**

> <article><ol><li>Run "mmctl team list" (or any command that requires communication with the server)</li><li>Run "mmctl team list --strict"</li></ol></article>

**Expected**

> <article><ul><li><strong>After 1</strong>:&nbsp;<ul><li>You will get feedback saying "ERROR: server version 5.xx.x.release-5.xx.xx.9bb0ce5.af1af0d71b40157b8025d9538a146623.true doesn't match with mmctl version 5.22.0. Strict flag is set, so the command will not be run"</li></ul></li></ul></article>
