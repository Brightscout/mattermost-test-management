## MM-T3320: Verified Create, attach and subscribe UI when additional instances are installed but not connected

### Details

| Name     | Value                    |
| :------- | :----------------------- |
| Priority | Low                      |
| Status   | Active                   |
| Folder   | Plugins / Jira / Jira V3 |
| Created  | 2020-08-21T13:55:07Z     |

### Steps

<hr/>

**Step 1.1**

> <article>1. Install the Jira plugin<br>2. Install one instance of Jira on your Mattermost server<br>3. Connect your test user</article>

**Expected**

> <article>A single jia instance is installed and connected<br><br></article>

<hr/>

**Step 1.2**

> <article>1. Use the slash command to install a 2nd Jira instance ex: /jira instance install cloud <a href="https://mmtest.atlassian.net/">https://mmtest.atlassian.net</a><br>2. Complete the installation by adding an application on the Jira side <br>3. Connect the test user to the 2nd Jira instance</article>

**Expected**

> <article>A send instance of Jira has been successfully added</article>

<hr/>

**Step 2**

> <article>1. Ensure your test user is only connected to 1 instance of jira<br>2. Click the ... menu on a Mattermost post<br>3. Select the Create Jira Issue option</article>

**Expected**

> <article>When the modal opens, no instance drop down is visible&nbsp;</article>

<hr/>

**Step 3**

> <article>1. Ensure your test user is only connected to 1 instance of jira<br>2. Click the ... menu on a Mattermost post<br>3. Select the Attach to Jira Issue option</article>

**Expected**

> <article>When the modal opens, no instance drop down is visible </article>

<hr/>

**Step 4**

> <article>1. Login as a System Admin<br>2. Ensure your test user is only connected to 1 instance of jira<br>3. Type /jira instance subscribe edit to open the subscription modal<br>4. Click Add Subscription</article>

**Expected**

> <article>The subscription creation page does not show the instance select drop down&nbsp;</article>
